◆ clock

Clock* libbgp::BgpConfig::clock

The clock to use for time-based events.

Time-based events like hold timer expired needs to refer to the clock. Sometime we may not want to use the system clock. For example, if BgpFsm is used inside a simulator like ns3, we will need to use the simulated clock instead of the real-time clock. Set this to NULL if you want to use a real-time clock.

Examples:
peer-and-print.cc, and route-event-bus.cc.

◆ nexthop

uint32_t libbgp::BgpConfig::nexthop

The default nexthop to use.

Default nexthop is used when sending routes to the peer. The nexthop value will remain unchanged if it is inside peering LAN. Default nexthop is used only when the nexthop attribute of an egress route is not in peering LAN.

◆ rev_bus

RouteEventBus* libbgp::BgpConfig::rev_bus

Pointer to the route event bus.

The route event bus is used to share information and communicate with other BGP FSMs. For example, route add/withdrawn events are sent to other FSMs with route event bus. Collision resolution also depends on the route event bus. You will need to create a route event bus object and pass it in as the configuration parameter for every FSMs. You may set route event bus to NULL if you are only using one FSM.

Examples:
peer-and-print.cc, and route-event-bus.cc.

◆ rib

BgpRib* libbgp::BgpConfig::rib

Pointer to the Routing Information Base object.

BGP FSM will use this RIB object to store routing information. If you would like to share RIB across different BGP FSMs, or pre-fill the RIB with some routes, you can create the RIB object yourself and pass it as configuration parameter here. If you set this to NULL, a new RIB will be created by BGP FSM. You can get it by calling BgpFsm::getRib.

Examples:
peer-and-print.cc, and route-event-bus.cc.
